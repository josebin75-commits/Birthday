# 🎁 HBD_LETTER: 나만의 웹 생일 편지 제작소
> 단 한 사람을 위해 퀴즈, 사진, 타자기 효과를 담아 특별한 웹페이지 생일 편지를 만들어주는 서비스입니다.

## ❓ 무엇을 하는 프로젝트인가
사용자(보내는 사람)가 웹페이지에서 친구의 이름, 맞춤형 퀴즈, 편지 본문, 그리고 추억이 담긴 사진을 입력하면, 파이어베이스(Realtime Database)를 통해 **고유한 생일 편지 웹 링크**를 생성해 줍니다. 
링크를 받은 친구는 스마트폰이나 PC로 접속하여 퀴즈를 풀고, 정답을 맞히면 감동적인 생일 편지를 읽을 수 있습니다. 또한, **편지 하단의 답장 폼을 통해 곧바로 고마움을 전하는 답장을 남길 수 있는 양방향 소통 기능**을 제공합니다.

## 🛠️ 사용 방법
1. **편지 생성하기**: `create.html` (제작 페이지)에 접속합니다.
2. **정보 입력**: 받는 사람의 이름, 둘만의 추억이 담긴 퀴즈와 정답, 편지 내용, 그리고 사진을 첨부합니다.
3. **링크 발급 및 공유**: [생일 편지 생성하기] 버튼을 눌러 발급된 고유 링크를 친구에게 전달합니다.
4. **편지 열람 및 답장**: 링크를 받은 친구는 퀴즈를 풀고 편지를 읽은 뒤, 하단의 입력창을 통해 답장을 작성하여 전송합니다.
5. **답장 확인**: 제작자는 파이어베이스 콘솔 DB(또는 답장 확인 링크)를 통해 친구가 남긴 답장을 실시간으로 확인할 수 있습니다.

## 📊 입력 / 출력

| 구분 | 입력 (Input) | 출력 (Output) |
| :--- | :--- | :--- |
| **텍스트 (생성)** | 친구 이름, 퀴즈 질문/정답, 편지 본문 | 화면 내 텍스트 렌더링 (이름에 따른 '아/야' 받침 처리, 타자기 효과) |
| **이미지 (생성)** | 로컬 사진 파일 (.jpg, .png 등) | Base64 문자열로 DB(`/letters`)에 저장 후 뷰어 화면에 렌더링 |
| **텍스트 (답장)** | 친구가 작성하는 답장 메시지 | 파이어베이스 DB(`/replies`)에 실시간 전송 및 저장 |
| **상호작용** | 퀴즈 정답 입력 및 [열어보기] 클릭 | 메인 편지 화면 전환 및 하단 폭죽(Confetti) 애니메이션 실행 |

## 🚧 아직 안 되는 것 (부숴보며 찾은 한계점)
이 프로그램을 직접 테스트하고 부숴보며 다음과 같은 한계점들을 발견했습니다.

1. **대용량 이미지 업로드 시 브라우저 멈춤 (Base64 한계)**
   - 파이어베이스 Storage 대신 Realtime Database를 사용하기 위해 이미지를 Base64 텍스트로 변환하여 저장하는 방식을 택했습니다.
   - **문제점:** 용량이 매우 큰 고화질 사진(수 MB 이상)을 업로드할 경우, 텍스트 변환 길이가 너무 길어져 로딩 화면에서 멈추거나 브라우저에 과부하가 걸리는 현상이 발생했습니다. 향후 이미지 리사이징(압축) 로직의 고도화가 필요합니다.
2. **답장 알림 기능의 부재 (UX 한계)**
   - **문제점:** 친구가 답장을 성공적으로 전송하여 DB에 저장되더라도, 편지 제작자에게 카카오톡이나 푸시 알림이 가는 시스템이 없습니다. 제작자가 직접 DB나 답장 확인 페이지를 열어보아야만 답장 도착 여부를 알 수 있는 한계가 있습니다.
3. **네트워크 및 캐시 의존성**
   - **문제점:** 간혹 브라우저 캐시 문제로 인해 예전 데이터가 보이거나 빈 화면이 뜨는 경우가 있었습니다. 또한, 모바일 데이터가 불안정할 경우 텍스트 인코딩이 깨지거나 애니메이션이 즉각적으로 실행되지 않을 수 있습니다.
